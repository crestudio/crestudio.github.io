VRChat 아바타 업로드 에러 해결법

2023년 7월 17일
마끼아또
VRSuya 모델러

요약​

  • Missing Script 컴포넌트 해결
  • 컴파일 에러난 프로그램 업데이트 또는 제거
  • VRCSDK 정상 상태 확인
  • 유니티 프로젝트 관리 기초 개념

VRChat 아바타가 업로드가 안 되어서 도움 요청 올라오는 케이스들을 읽어 보면 대부분 여기 가이드의 방법을 사용하면 해결되는 경우가 많음

이 외에 화이트리스트에 없는 컴포넌트를 사용하고 있거나, 사운드 에셋 설정을 누락했다던가 이런 조건들은 보통 VRCSDK 패널에서 메시지가 나오니 생략


Missing Script 컴포넌트 해결​

Missing Script가 있으면 업로드 눌러도 뭔가 진행하다가 아무 일이 안 일어나는 경우가 많음

아니면 Build & Publish for Android 같이 윈도우 용 Unity 프로젝트인데 퀘스트 용으로 빌드하려는 버튼으로 변하는 경우도 있음

현재 대부분 모듈러 아바타 컴포넌트가 포함된 의상과 기믹들이 올라오는데 해당 컴포넌트를 설치하지 않은 경우에는 업로드가 불가능함

구형 의상이나 아바타 경우에는 주로 Dynamic Bone(다이나믹 본) 시절 컴포넌트가 남아있어서 그러는 경우가 많음

해결법은 무슨 컴포넌트인지 알면 해당 애드온을 설치하거나 Missing Script 컴포넌트를 삭제하는 방법인데, 어떤 애드온이 필요한지 알려면 해당 에셋의 페이지를 참고하면 됨

Missing Script 컴포넌트를 제거하는 애드온은 lilAvatarUtils(삭제) 애드온이 있고, 여기도 관련해서 Cleaner(선택+삭제)라는 애드온을 만들었는데 편한 쪽으로 쓰면 될 듯


컴파일 에러난 스크립트 제거​

그림처럼 프로젝트를 열고 Console 패널에서 Clear 버튼을 누르면 아무것도 안 남아야 하는데, 화면처럼 오류가 계속 남는다면 프로젝트에 설치된 어떠한 스크립트에서 컴파일 에러가 난 거임

해결 방법은 두 가지인데

  • 오류가 발생하는 애드온 패키지를 최신 버전으로 업데이트
  • 오류가 발생하는 애드온 패키지를 프로젝트에서 완전 삭제

이 때 업데이트를 하거나 삭제 할 때, 덮어쓰기하거나 일부만 삭제하지 말고 애드온 전체 폴더를 업데이트하거나 삭제해야 함

대부분 애드온은 내용물이 서로 연결되어 있고, 특히 애드온을 업데이트 할 때 파일이 추가/제거 되는 케이스가 많은데 덮어쓰기 하면 기존 내용과 최신 내용이 뒤죽박죽 섞여서 컴파일 에러가 나는 경우도 있음, 심지어 똑같은 내용을 가진 애드온을 두 개 이상 설치해도 컴파일 에러가 남

그래서 애드온을 자동으로 폴더 째로 원클릭 업데이트 해 주고 관리하기 편하라고 VCC에 패키지 관리자 기능이 있으니 그걸 지원하는 애드온을 쓰는게 좋음

마시멜로 피직스본 같이 VCC로 업데이트를 할 수 없는 유료 애드온인 경우나, 내부에 뭔가 데이터를 저장하는 특수 케이스는 폴더를 통째로 백업하고 삭제한 다음 재설치하는 방식으로 진행하면 문제가 생기더라도 해결하기 좋음

보통 위의 작업을 하면 해결 될 거고, 업데이트 해도 너무 오래되어서 최신 VRCSDK를 지원 안 한다면 비슷한 툴을 새로 찾거나 포기하는게 좋음


VRCSDK 정상 상태 확인​

위의 스크린샷과 비슷하게 VRCSDK가 깨져서 아바타 디스크립터가 제대로 안 나오거나, VRCSDK 메뉴 아래에 아무것도 없다 이런 경우에는 VRCSDK를 재설치해야 함, 위의 컴파일 에러 때문에 같이 깨지는 케이스도 있어서 해결하면 VRCSDK가 복구되는 경우도 있지만 개인적인 경험으로는 VRCSDK가 깨지는 경우에는 프로젝트를 새로 파야 해결되는 경우가 많았음

주로 VRCSDK 버전 변경으로 이렇게 되는 경우가 많은데, 임시 데이터가 남아서 그런지 패키지를 지우고 깨끗하게 설치해도 여전히 깨지는 케이스가 많아서 이런 경우에는 유니티 종료하고 프로젝트 폴더의 Library 폴더를 통째로 날리고 다시 켜 봐야 함


위의 Library 폴더를 삭제하고 유니티를 켜면 처음부터 스크립트들을 컴파일하고 모든 에셋을 다시 다 유니티 게임용으로 불러오는 작업을 하게 됨

이렇게 해도 안 되는 경우에는 프로젝트를 새로 파는게 제일 빠름

정보

Unity 프로젝트를 백업할 때 거의 대부분 유저들은 Assets 폴더만 백업하면 나머지는 인터넷에서 구하는 데이터들이므로 이 방식으로도 프로젝트를 복구 할 수 있다, 따라서 시간과 용량 절감을 위해서 Assets 폴더만 백업하자


프로젝트 새로 파는 방법​

VCC에서 새로운 프로젝트를 만든다 → 해당 유니티 프로젝트를 연다 → 유니티에서 VRCSDK 패널 열어서 제대로 열리는지 에러가 없는지 확인 → 쉐이더 임포트 → 에러 없는지 체크 → 아바타 데이터만 옮기기 → 에러 없는지 체크 → 애드온 각각 하나씩 설치

특히 에러난 프로젝트에서 복구하는 경우에는 위의 순서를 반드시 지켜서 복구하자, 어디서 문제가 생기는지 알면 문제 해결하기가 쉽고 그리고 도움을 요청할 때에도 어느 과정에서 에러 났는지 알려주면 답변 해 주기도 좋다

어디서 에러가 났는지 확인하려면 번거롭겠지만 꼭 이렇게 복구하는게 좋다

완전 아무 것도 없는 빈 프로젝트에서 VRCSDK도 안 되는거는 다운로드 오류이거나 진짜 컴퓨터 문제일 수 있다

경고

VCC를 통하지 않고 유니티 프로젝트를 만드는 경우, VRCSDK가 컴파일 완료가 되어야 의존성 애드온들이 제대로 컴파일 되므로 데드락을 피할려면 반드시 순서를 지켜야 한다


쉐이더 및 애드온 관리​

VRChat 커뮤니티가 커지면서 각종 쉐이더와 애드온도 많아진 상황인데, 이러한 프로그램들이 많아질 수록 유니티 프로젝트도 무거워지고 문제가 생겼을 때 해결하기 어려워짐

정리하자면 쉐이더는 가능한 1가지 종류로 통합해서 줄이고, 안 쓰는 애드온은 과감히 삭제하면 됨

극단적으로 기본 쉐이더가 되겠지만 VRCSDK랑 아바타만 있어도 업로드가 가능함, 다시 말해서 다른 애드온들은 필요에 의해서 존재해야 하는거지, 필요 없다면 지우는게 문제를 일으킬 소지가 적어지고 고질적인 유니티 튕김도 방지 할 수 있음

왜냐하면 프로젝트에 존재하는 프로그램들은 전부 다 컴파일 되어서 메모리에 계속 존재하는데 유니티에서 뭔가 작업 할 때마다 조건이 맞으면 트리거가 되어서 실행 될 수 있는데 이 때 뭔가 안 맞으면 유니티가 튕기거나 오류가 생긴다고 생각하면 편할 듯

어차피 필요하면 다시 깔면 되니까, 쓰고 나서 안 쓰는 애드온은 과감하게 지우자

일 때문에 쉐이더를 여러 개 깔아놓고 쓰는데 한 개로 통일해서 쓸 때에는 한 번도 안 튕기던 유니티가 쉐이더 여러 개 설치되고 나니 자꾸 크래시 남
