Unity Prefab 가이드

April 7, 2025
마끼아또
VRSuya 모델러

요약​

  1. Variant Prefab 기능을 잘 활용하자
  2. Prefab을 잘 활용하면 제작자 뿐만 아니라 소비자들도 유용하다
  3. 그렇지 않으면 노가다해서 정상화해야 한다

서론​

옛날 VRChat은 아바타를 구매해도 모델이랑 텍스쳐 파일만 달랑 들어 있었던 시절이 있었다, 그 때 당시 VRChat 유저라면 머테리얼이나 다이나믹 본, 표정 애니메이션 제작, VRChat 아바타 세팅을 모두 직접 할 줄 알아야 살 수 있었던 정말 야생의 시대였다, 하지만 시간이 어느덧 흘러 많은 제작자들이 아바타 세팅을 미리 해 주고 기믹이 있는 의상이라도 이제 넣기만 해도 입힐 수 있는 시대가 도래했으니, 이러한 모든 것들이 가능해진 이유가 바로 프리팹(Prefab) 이라는 개념이 존재하기 때문이다


프리팹(Prefab) 개념​

간단하게 표현하자면 템플릿이랑 비슷한 거라고 보면 된다, 아바타를 샀는데 VRChat 아바타 세팅하고 피직스본 세팅해야 되고, 옷을 샀는데 기믹 세팅해야 되는 이런 상황을 매번 하고 싶어하는 사람은 없을 것이다, 아바타 세팅 다 되어있는 완성품이 있으면 당연히 편할 것이다, 이러한 역할을 해 주는 게 바로 프리팹이다

유니티에서 직접 느껴보고 싶다면 아바타 모델 원본 파일을 씬에다가 갖다 놓아보자, 정말 아무런 기능이 안 들어있어서 머리카락 하나 안 움직이는 모습을 볼 수 있지만, 프리팹으로 해 본다면 이러한 설정들이 다 세팅되어 있다는 차이점을 쉽게 발견할 수 있다

아래에는 유니티에서 사용되는 프리팹의 4가지 종류이다

위의 내용들은 간단하게만 이해하고 아래 내용들을 읽을 때 이러한 프리팹들이 사용되는구나 하고 넘어가면 된다


위의 아바타 5명을 보면 거의 다 똑같은데 옷만 서로 다르다는 거를 볼 수 있다, 이거를 구현하려면 몇 가지 방법이 있다

  • 첫번째로는 한 아바타에 옷을 모두 넣고 옷장 기능을 쓰는 방식
  • 두번째로는 한 아바타 베이스로 옷을 각각 넣은 바리언트 프리팹을 만드는 방식

난이도랑 접근성으로는 당연히 첫번째가 편할 것이다, 아바타 하나에 옷을 다 넣고 옷장을 만들어서 올리면 VRChat에서 버튼 하나 누르면 옷도 바뀌고 로딩도 없으니까, 이렇다 보니 대다수 유저들이 아바타 하나에 옷을 다 넣어서 최적화가 어려워지고 점점 게임이 무거워지고 있다, 현실로 비유하자면 내가 당장 안 입고 있는 옷을 가방에 넣어두고 다른 사람들에게 계속 들고 있게 하는거랑 비슷하다고 봐도 된다

두번째 방식은 어떨까? 합쳐 놓은걸 개별로 바꿔야 하니 아래와 같은 점들이 있다

  • 의상마다 업로드를 해야 하니, 아바타 슬롯이 늘어나고 업로드 합산 시간이 조금 길어진다
  • 제대로 된 Prefab 방식으로 설계 안 해 두었으면 관련 세팅들을 맞춰서 바꿀 필요성이 있다
  • 원본을 보존하고 개별로 해야 되니 추가적인 자식 역할을 할 파일을 생성 및 관리해야 된다

프리팹에 대한 자세한 개념과 사용 방법은 아래에서 조금 더 설명하고, 의상마다 업로드하기 귀찮은 문제는 Continuous Avatar Uploader 라는 애드온을 사용하면 해결이 된다, 업로드 눌러놓고 유투브 보고 있으면 됨

자 그럼 프리팹에 어떤 기능이 있는지 알아보고 제대로 사용 방법을 배워보자


원본 Prefab과의 차이점​

먼저 편리하고 강력한 기능 중에 하나가 바로 원본을 기반으로 무엇이 달라졌는지 알기가 쉽고, 변경 사항을 적용하거나 되돌리는게 간편하다는 점이다

원본 아바타랑 비교했을 때, 가슴이 커지고 오피스룩을 입고 있고 안경을 쓰고 있고… 이러한 변경 사항들이 있다는 걸 쉽게 파악할 수 있다

이러한 Prefab들을 만들다 보면 이 화면에서 내가 건드리지 않았는데 변경되어 있는 것들이 의외로 많다는 걸 알게 되는데, 그런 걸 선택해서 되돌리고 원하는 사항만 남기면서 전체적인 구조를 파악해 두면 아바타 관리하는게 편해지는 걸 체감할 수 있다


하나씩 눌러보면 어떠한 점들이 구체적으로 바뀌었는지 왼쪽과 비교해서 볼 수 있는데, 여기에서 통째로 아니면 빨간 상자처럼 부분부분만 또 나눠서 액션을 취할 수 있다

원하지 않으면 되돌리면 되고, 상위 프리팹에 적용한다면 다른 바리언트 프리팹에도 전파하여 적용 시킬 수도 있다


Prefab 변경사항 적용 및 되돌리기​

프리팹 내용 중에서 가장 핵심인 기능들이다

Prefab을 씬에 올려놓고 수정하면 옆에 파란색 줄으로 변경사항이 있다고 표시가 되는데, 여기서 글씨나 헤더 부분을 오른쪽 클릭하면 관련 메뉴가 나온다

Apply는 변경사항을 프리팹에 적용을 시켜준다, 이 때 어느 프리팹에 적용시키는게 따라서 결과가 달라지니 어디에 적용하는지 잘 읽어보자

Revert는 변경사항을 취소하고 원본 내용으로 다시 바꿔준다

여기서 중요한 포인트는 파란색은 오버라이드(Override) 상태라고 하여 ‘나는 원본과 내용이 달라져야 해요’ 라는 뜻이기 때문에, 오버라이드 상태이면 원본과 같은 값이더라도 원본이 바뀌어도 하위 값은 변하지 않는다, 다시 말해 원본을 따라가야 하는 내용이라면 Revert를 통해 다시 오버라이드를 취소할 필요가 있다


도식화를 하면 위와 같은 개념으로 적용이 된다, 어려우면 직접 유니티에서 오브젝트를 만들어서 테스트해 보면 이해가 더 빠를 것이다

하위 프리팹(Variant Prefab)에서 상위 프리팹으로 적용하면 파란색이 없는 애들에게만 전파가 되는 모습을 볼 수 있다

따라서 프리팹 기능을 잘 활용하려면 변경 사항을 관리하고 위에 들어갈 내용인지 아래에 들어갈 내용인지 구분하는게 포인트라 할 수 있다

예를 들면 상위 프리팹에는 AFK라던가 기믹이라던가 공통적으로 적용하고 싶은 내용들을 넣으면 하위 프리팹에도 자동으로 적용 되어서 편할 것이고, 반대로 하위 프리팹에는 옷이라던가 색상 머테리얼 같이 유니크한 변경 사항들이 존재할 것이다

그래서 아바타 옷을 입히다 보면 부모가 되는 프리팹을 잘 만들어두면 생각 외로 하위 프리팹에는 넣을 게 그다지 없어서 금방 만들게 된다, 또한 모듈러 아바타가 왜 모듈인지 진가를 몸소 체감 할 수 있다, 넨도로이드처럼 필요한 프리팹만 쏙쏙 넣으면 바로 완성되므로 프리팹과 모듈러 아바타의 시너지가 굉장하다는 걸 알 수 있다


추천하는 Prefab 구조​

위의 내용들을 바탕으로 추천하는 방식은 위의 도식화랑 같음, 이런 느낌에서 원한다면 늘리고 줄이고 하면 됨

이렇게 프리팹으로 관리하면 아래와 같은 점들이 있다

  • 아바타가 업데이트 되더라도 내가 수정한 부분이 날아갈 우려가 적음, 이건 아바타 제작자들도 업데이트할 때 이런 개념을 잘 이해해야 되는 부분이기도 함, 왜냐하면 아바타 제작자는 부모 Prefab 역할도 동시에 하기 때문에 잘 관리 할 수 있어야 구매자들도 업데이트 할 때 받아들이기 쉽고 적용하는데 부담감도 적어짐
  • 적은 수정으로 모든 아바타에 적용 가능, 주로 기믹 쪽에 해당 되는데, 아예 다른 아바타라도 프리팹으로 공통된 요소를 관리했다면 한 번만 작업하면 일괄 적용됨
  • 유니티 씬 파일이나 일부 최하단 프리팹이 손상되어도 문제 없음, 프리팹은 따로 파일로 관리되기 때문에 씬 파일이 날아가도 도식화처럼 관리 했으면 사실 사라진 내용이 없기에 금방 복구 가능, 하위 프리팹 쪽도 당연히 상단 프리팹에 비해 들어있는 내용이 적으니 복구하는데 시간도 적게 걸림

Prefab 만들기​

앞에서 많은 이야기를 했던거에 비해 프리팹을 만드는건 아주 간단하다, 그냥 하이어라키에 있는 게임 오브젝트를 쭈욱 잡아서 프로젝트 폴더로 끌어다 놓으면 위와 같이 물어본다

여기서 특별한 목적이나 이유가 있는게 아닌 한 Prefab Variant를 눌러서 만들면 된다, 그럼 상단 프리팹과 계속 연결이 유지되어 아까와 같은 혜택을 볼 수 있다

만드는건 어렵지 않고 앞서 배웠던 프리팹과의 관계와 구조를 설계하고 변경사항들을 잘 관리하는게 더 중요한 포인트이기 때문에 그렇다

만약 첫번째 오리지널 프리팹을 만들게 된다면 어떤 일들이 벌어지게 되는걸까?


위와 같이 오리지널 프리팹은 정말 독립적인 최상위 부모와 같은 존재가 되므로 다른 요소에 어떠한 영향을 받지 않게 된다

여기서 주목해야 되는 점은 두번째 줄에서 모델 프리팹은 오리지널 프리팹이 되어버린 하위 프리팹에게 영향을 주지 못한다는 걸 볼 수 있다

옛날 BOOTH 아바타들은 저런 식으로 오리지널 프리팹으로 생성을 하였기에, 대형 모델 패치를 해 버리면 아예 새로 작성을 해서 배포를 해야 했다

소비자들도 지금 겪을 수 있는게 모델 파일을 패치했는데 아바타가 사라졌어요! 같은 내용이 대표적인 사례이다, 그래서 VRChat 강좌들 중에 ReassignBoneWeigthsToNewMesh 와 같은 스크립트를 활용해서 깨지거나 안 보이는 아바타를 복구하고 그랬던 이유가 바로 이러한 문제가 있기 때문이다


아바타 및 의상 제작자를 위한 프리팹 내용​

의외로 제작자들 중에서도 이러한 내용을 몰라서 모델 Prefab 기반인데도 Original Prefab으로 해 두는 케이스가 정말 많음

제작자들이 어려워 하는 파트 중에 하나가 유지보수 분야인데, 이러한 개념들을 이해하고 적용해 두면 모델을 수정하거나 다양한 바리언트가 있어도 한 번만 잘 수정하면 되니까, 수정에 대한 부담감이 줄어들어서 편함

특히 아바타 제작자라면 Variant Prefab으로 배포하는게 중요함, 요즘 페이셜과 같은 모델을 수정하는 관련 상품들도 많이 판매 되는데 모델 파일을 수정하면 Unity에도 제대로 반영이 되니까 구매자들이 버그 없이 적용하기 좋아짐

반대로 말하자면 제작할 때에 FBX 모델 파일을 수정을 많이 하게 되는데, 매번 다시 Unity 세팅하는 것보다 FBX 파일을 덮어씌워도 되니까 훨씬 빠르게 작업을 진행을 할 수 있어서 편함, 그렇지 않으면 아래의 방식으로 매번 복사 해 줘야 하는데 은근 값이 달라질 때도 있어서 오히려 덮어씌우는게 더 빠르고 안정적임, 휴머노이드 본을 수정한 경우에는 약간의 트릭도 필요한데 여기 주제랑은 안 맞으니 생략

기존 Original Prefab을 변환하는 방법은 모델 Prefab에서 처음부터 다시 시작해서 재구축하는 방법이 제일 깔끔함, 당연히 하나하나 다시 하기 번거로우니까 Pumkins Avatar Tools 애드온을 활용하여 변경사항들을 그대로 복사 해 주는 기능이 있으니 이걸 쓰면 편함, 그리고 위의 내용들을 바탕으로 필요 없는 내용 다시 되돌리고 꼭 필요한 내용만 남기면 깔끔하게 완성


후기​

은근 제작자들도 잘 몰라서 이러한 내용을 설명 해 줄 때도 많고, 관련해서 생기는 문제들로 사람들이 질문 글을 올리는 걸 간간히 봐서 사람들이 알았으면 좋겠다 싶은 개념이라서 적어봤음
